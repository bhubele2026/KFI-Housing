Add comprehensive export to Excel and PDF for the HousingOps assistant.
Operators should be able to say "export X to Excel" or "give me a PDF of
Y" for ANY listable data in the app — leases, occupants, beds, properties,
payroll deductions, utilities, insurance certificates, room-nights — and
get a downloadable file chip in the chat. Filters that work in the
matching list_* read tool also work in the export tool.

CRITICAL: Excel files must use LIVE FORMULAS for totals, ratios, deltas,
percentages — never hardcode computed values. This applies to every
total row, every monthly/weekly aggregate, every variance column. If a
cell could conceivably be a formula, make it a formula.

DATABASE — assistant_exports table
═══════════════════════════════════════════════════════════════════════════

Drizzle migration. Mirror the pattern of assistant_uploads:

  id           text primary key            -- "ax-xxxxxxxx"
  userId       text not null
  conversationId text                       -- null if export was triggered outside chat
  filename     text not null               -- "leases-burnett-hinckley-2026-05-27.xlsx"
  mime         text not null
  sizeBytes    integer not null
  content      bytea not null
  toolName     text not null               -- which export tool produced it
  format       text not null               -- "xlsx" | "pdf"
  entityType   text not null               -- "lease" | "occupant" | "bed" | etc.
  rowCount     integer not null
  filterDesc   text                         -- human-readable filter summary
  createdAt    timestamptz default now()
  expiresAt    timestamptz not null        -- createdAt + 24h

  Indexes:
    (userId, createdAt desc)
    (expiresAt)

Cleanup job: delete rows where expiresAt < now() every hour. Use the same
scheduling primitive the rest of the app uses. Do NOT add a new scheduler.

SHARED EXPORT BUILDERS
═══════════════════════════════════════════════════════════════════════════

xlsx-export.ts — shared types and one entry function:

  type CellFormat = "text" | "currency" | "date" | "number" | "percent" | "int";

  interface ExportColumn {
    header: string;
    key: string;                          // dot-path into the row object
    width?: number;
    format?: CellFormat;
    /** If set, this column emits a per-row formula instead of a value.
     *  rowIndex is 1-based and refers to the data row (header is row 1
     *  inside the worker; offset internally). */
    formula?: (rowIndex: number) => string;
  }

  interface SummarySheet {
    label: string;                        // shown in row 1 of summary sheet
    rows: Array<{
      label: string;
      formula: string;                    // e.g. "=SUM(Data!D2:D{lastRow})"
    }>;
  }

  export async function buildXlsxBuffer(opts: {
    title: string;                        // workbook title shown in row 1 of main sheet
    filterDesc: string;                   // shown in row 2 of main sheet
    columns: ExportColumn[];
    rows: Array<Record<string, unknown>>;
    summary?: SummarySheet;               // optional second sheet with totals
    freezeHeader?: boolean;               // default true
  }): Promise<Buffer>

  Implementation requirements:
    - Row 1: title (merged across columns), bold, larger font
    - Row 2: filterDesc (italic, gray)
    - Row 3: empty
    - Row 4: column headers (bold, bottom border)
    - Row 5+: data
    - Apply number format per column based on `format`:
        currency → "$#,##0.00;[Red]-$#,##0.00"
        date     → "yyyy-mm-dd"
        percent  → "0.0%"
        number   → "#,##0.00"
        int      → "#,##0"
    - Autosize columns to fit content (cap at 50 chars wide)
    - Freeze panes at row 5 if freezeHeader
    - Apply table styling (banded rows)
    - Summary sheet, if provided, has its own title row and total rows;
      formulas reference the data sheet using "Data!" prefix
    - {lastRow} placeholder in formulas gets replaced with the actual
      last data row index before write

pdf-export.ts — same data, simpler output:

  export async function buildPdfBuffer(opts: {
    title: string;
    filterDesc: string;
    columns: ExportColumn[];              // formula columns rendered as their computed value in PDF
    rows: Array<Record<string, unknown>>;
    summary?: SummarySheet;               // rendered as a totals block at the bottom
  }): Promise<Buffer>

  Implementation requirements:
    - Letter, landscape, 0.5in margins
    - Header on every page: title (left), generation timestamp (right)
    - Footer on every page: "Page N of M" (centered)
    - Column headers repeat on each page
    - Currency / date / percent formatted using the same rules as xlsx
    - For formula columns in PDF, COMPUTE the value at export time and
      render the number. PDFs don't recalc.
    - If rows exceed what fits, wrap to next page; do NOT shrink text
      below 8pt — instead, if a column would be too narrow, drop the
      least-important columns and add a note "(N columns omitted — use
      Excel export for full data)"

TOOL DEFINITIONS — eight export tools, all read-only
═══════════════════════════════════════════════════════════════════════════

In tools.ts, add EIGHT new tools, all kind="read" (exports don't mutate
state — no confirm card needed, just execute). Each tool calls the
matching list_* logic internally so filters stay consistent.

Common input shape on every export tool:

  format: "xlsx" | "pdf"        (required)
  filename?: string             (optional override; default auto-generated)
  ...same filter args as the matching list_* tool...

Common return shape:

  { exportId: string, filename: string, format: string,
    rowCount: number, sizeBytes: number }

The client renders this return as a download chip (see UI section).

Tools to add:

  1. export_leases
     description: "Export leases to Excel or PDF. Accepts the same filters
        as list_leases (customerId, propertyId, status, expiringWithinDays,
        snoozed). Returns a download chip the operator can click."
     columns (xlsx):
       Lease ID | Property | Building | Unit | Start | End | Status |
       Monthly Rent | Security Deposit | Vendor | Notes | Days to Expiry
     formula columns:
       Days to Expiry → =IF(F{r}="", "", F{r}-TODAY())
       (where F is the End column — index dynamically based on column order)
     summary sheet (Totals):
       Total leases           → =COUNTA(Data!A5:A{lastRow})
       Active leases          → =COUNTIF(Data!G5:G{lastRow},"Active")
       Expiring next 30d      → =COUNTIFS(Data!L5:L{lastRow},">=0",Data!L5:L{lastRow},"<=30")
       Monthly rent total     → =SUM(Data!H5:H{lastRow})
       Avg monthly rent       → =AVERAGE(Data!H5:H{lastRow})

  2. export_occupants

     columns:
       Occupant ID | Name | Property | Bed | Company | Status |

       Total occupants, Active, Former, Total weekly charges (SUM)

  3. export_beds

  4. export_properties

Naming rule for the file: `<entityType>-<scopeName?>-<YYYY-MM-DD>.<ext>`.
Example: "leases-burnett-hinckley-2026-05-27.xlsx". If a scope name can't
be cleanly resolved, fall back to "<entityType>-<YYYY-MM-DD>.<ext>".

Each export tool, after generating the buffer, inserts an
assistant_exports row with userId from ctx, conversationId from the
calling route, and the buffer in `content`. Return the metadata, NOT
the buffer (we never send the bytes back through the model — they
download via a separate HTTP endpoint).

DOWNLOAD ENDPOINT (exports.ts)
═══════════════════════════════════════════════════════════════════════════

New route file. Add ONE endpoint:

  GET /api/assistant/exports/:id/download

  - Validate the export row exists and belongs to req.user.id (same auth
    middleware as the rest of /api/assistant/*)
  - Validate not expired (expiresAt > now); if expired, 410 Gone with
    body { error: "Export expired — please regenerate." }
  - Set headers:
      Content-Type: row.mime
      Content-Disposition: attachment; filename="<row.filename>"
      Cache-Control: private, no-store
  - Stream row.content as the body

NO listing endpoint. NO delete endpoint. Exports are ephemeral; the
cleanup job handles eviction.

CLIENT — render the download chip
═══════════════════════════════════════════════════════════════════════════

In use-assistant.ts:

  1. Extend the SSE `tool_result` event handler. Currently the client
     ignores tool_result for reads. For export tools specifically, the
     server should ALSO send a new SSE event after the tool result:

       sseSend(res, "export_ready", {
         exportId, filename, format, rowCount, sizeBytes,
         downloadUrl: `/api/assistant/exports/${exportId}/download`
       });

     Add this in index.ts's runLoop right after a successful read-tool
     execute, by checking if the tool name starts with "export_" and the
     result has an exportId. Don't wedge export-specific logic into the
     generic tool_result event — keep them separate.

  2. Add `exports: ExportChip[]` to InternalState:

       interface ExportChip {
         id: string;          // exportId
         filename: string;
         format: "xlsx" | "pdf";
         rowCount: number;
         sizeBytes: number;
         downloadUrl: string;
         downloadedAt?: number;   // local-only, set when user clicks
       }

  3. Append to exports on the export_ready event.

In assistant-bubble.tsx:

  4. Render each export chip as a card in the message stream, in the
     position it was generated (alongside the matching assistant turn).
     Card structure:

       [file icon for xlsx/pdf]  filename
       N rows · ~size           [Download]

     Clicking Download triggers a same-tab fetch with credentials and
     saves the blob via a temporary <a download> click — do NOT use
     window.open(downloadUrl) because that loses the auth cookie on some
     browsers. Implementation:

       const r = await fetch(chip.downloadUrl, { credentials: "include" });
       const blob = await r.blob();
       const url = URL.createObjectURL(blob);
       const a = document.createElement("a");
       a.href = url; a.download = chip.filename; a.click();
       URL.revokeObjectURL(url);

     Mark downloadedAt locally so the chip dims slightly after click
     (still re-downloadable).

  5. Format size: "12 KB", "1.4 MB". Format rowCount: "127 rows".
